# Implementation Plan: Maven Central Publishing & CI/CD Pipeline

**Branch**: `004-maven-publish-cicd` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/004-maven-publish-cicd/spec.md`

## Summary

Configure the jsf-autoreload multi-module Maven project for publishing to Maven Central via the Sonatype Central Portal, and establish GitHub Actions CI/CD pipelines for automated building, testing, and releasing. This involves: adding a `release` Maven profile with source/javadoc/GPG-signing/Central-publishing plugins, creating a CI workflow (JDK 11+21 matrix on PRs and pushes to main), and a release workflow triggered by version tags that deploys atomically to Central and auto-bumps to the next SNAPSHOT version.

## Technical Context

**Language/Version**: Java 8+ (compile with `--release 8`); CI matrix: JDK 11 + JDK 21  
**Primary Dependencies**: JSF API (javax.faces 2.3 / jakarta.faces 3.0+), Servlet API 3.0+, Maven Plugin API — all `provided` scope. New build-only deps: `central-publishing-maven-plugin:0.10.0`, `maven-source-plugin:3.3.1`, `maven-javadoc-plugin:3.11.2`, `maven-gpg-plugin:3.2.7`  
**Storage**: N/A  
**Testing**: JUnit 5 + Mockito + maven-surefire-plugin + maven-failsafe-plugin  
**Target Platform**: JVM (Java 8+), GitHub Actions (ubuntu-latest)  
**Project Type**: Library (multi-module Maven project — 3 publishable modules + 1 test module)  
**Performance Goals**: CI feedback under 10 minutes per PR  
**Constraints**: Sonatype Central Portal validation rules (see [research.md](research.md) R-001, R-006)  
**Scale/Scope**: 4 modules, 3 publishable; 2 GitHub Actions workflows
